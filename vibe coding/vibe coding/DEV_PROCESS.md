# 开发全过程（AI协同）说明

## 1. 开发方式
- 使用本地 VS Code + GitHub Copilot（GPT-5.3-Codex）进行 vibe coding。
- 策略：先跑通主干功能，再做需求变更与回归。
- 目标：尽量少手写，优先由 AI 生成代码、文档、测试草稿。

## 2. 迭代过程
1. 建立 Flask + SQLite 项目骨架。
2. 实现登录、日程 CRUD、提醒接口。
3. 完成前端页面与交互（新增/编辑/删除/完成）。
4. 完成本地部署与演示文档。
5. 响应需求变更：实现 0级提醒弹窗（强提示、必须确认）。
6. 响应需求变更：实现可视化日历（颜色高亮时间段）。
7. 执行回归测试并记录结果。

## 3. AI协同细节
- **【核心规约】**：**所有 AI 代码生成与重构行为必须严格遵循项目根目录下的 [CLAUDE.md](CLAUDE.md) 规则**。如发生冲突，以 `CLAUDE.md` 规则为最高优先级。
- AI负责：
  - 代码编写前需明确申明假设及目标动作，按 `CLAUDE.md` 进行“先思考后执行”。
  - 路由与前端交互代码生成。
  - 样式风格调整与响应式适配。
  - 遵守最简代码原则（Simplicity First）确保不加无用代码及过度封装。
- 人工负责：
  - 验收规则确认。
  - 演示流程把关。
  - 对关键行为（提醒策略、时间显示）做人工验证。

## 4. 质量保障
- 每轮改动后做静态错误检查。
- 关键改动后执行 `pytest -q`。
- 需求变更后执行回归清单，避免影响原有 CRUD 与提醒。
