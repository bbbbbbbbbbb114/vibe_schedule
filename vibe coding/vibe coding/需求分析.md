# 1. 项目背景与目标
本项目旨在通过“vibe coding”方式，开发一个面向用户的日程管理Web应用。用户可进行日程的增删改查，并在页面打开时获得实时提醒，登录时可查看过期提醒。开发过程中强调与AI协同，减少手写代码。

# 2. 功能需求
1. 用户登录/注册（可选，便于多用户扩展）
2. 日程管理
   - 新增日程
   - 查看日程列表
   - 编辑日程
   - 删除日程
3. 日程提醒
   - 页面打开时实时提醒即将到期/已到期日程
   - 登录时显示过期提醒
4. 前端页面
   - 严格按照[Design.md]要求的风格生成
   - 简洁的日程管理界面
   - 实时提醒弹窗或高亮
5. 后端API
   - 支持日程的CRUD操作
   - 支持提醒逻辑

# 3. 技术选型理由
1. IDE选择：VS Code
   - 理由：本地开发体验优秀，插件丰富，AI集成（如Copilot）便捷，调试与部署友好，适合Python全栈开发。
2. Python开发框架：Flask
   - 理由：轻量、易用，适合快速开发Web应用，社区资源丰富，原生支持SQLite，便于与AI协同生成代码。

# 4. 非功能需求
- 数据存储：SQLite，便于本地开发和演示
- 部署：本地运行，支持演示
- 测试：单元测试+手动功能测试
- 代码协同：最大化利用AI生成代码，减少手写
- 文件规模: 不超过2000行

# 5. 主要开发流程
1. 需求分析与技术选型
2. 数据库模型设计（用户、日程表）
3. 后端API设计与实现
4. 前端页面设计与实现
5. 日程提醒机制实现
6. 本地部署与演示
7. 测试（单元测试、功能测试）
8. 需求变更响应与回归测试

# 6. 关键点说明
- 实时提醒：可用WebSocket或前端定时轮询实现
- 过期提醒：用户登录后查询并展示
- AI协同：需求、代码、测试用例均优先由AI生成，人工只做微调
