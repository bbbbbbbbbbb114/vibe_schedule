# 测试与回归测试说明

## 1. 测试范围
- 日程 CRUD：新增、查询、编辑、删除、完成状态切换。
- 提醒功能：
  - 登录后过期提醒。
  - 页面打开时实时提醒。
  - 提醒配置：提前 5 分钟/10 分钟/1 天。
  - 提醒时机：开始/结束/都提醒。
  - 0级弹窗提醒：必须点击确认才关闭，声音单次播放。
- 重复规则：不重复/每天/每周固定日。
- 地点标签：选填，列表与日历详情可见。
- 可视化日历：
  - 时间点/时间段按类型展示。
  - 周期事件实例展开显示。
  - 当前时间红线显示。
  - 不同日程颜色高亮。
  - 点击日历事件展示详情。

## 2. 自动化测试
执行命令：

```powershell
pytest -q
```

当前自动化覆盖：
- 应用创建与 SQLite 配置验证。
- 登录 + 日程 CRUD + 提醒接口基础校验。
- 时间段创建与查询。
- 提醒配置 + 重复规则（daily/weekly）触发校验。

## 3. 手动测试用例
1. 登录后新增一条“时间点”日程，检查列表与日历均出现。
2. 新增一条“时间段”日程，检查日历块长度符合开始-结束时段。
3. 新增“每天”重复日程，检查日历未来多天出现实例。
4. 新增“每周固定日”重复日程，检查仅对应周几出现。
5. 填写地点，检查列表标签与日历详情展示地点。
6. 配置提醒提前量和时机，验证弹窗在预期窗口触发。
7. 检查日历红色当前时间线是否显示。
8. 编辑标题后，列表与日历标题同步更新。
9. 删除日程后，列表与日历同时移除。
10. 勾选完成后，列表样式置灰，日历颜色变灰。
11. 新建即将到期日程，等待轮询触发 0级弹窗；确认声音只播放一次。
12. 不点击“我已确认”时弹窗不消失；点击后关闭。
13. 新建过期日程，刷新后出现在“过期提醒”。

## 4. 回归测试清单（需求变更后）
- R1：登录/退出流程正常。
- R2：时间录入与展示无时差偏移（例如 18:22 仍显示 18:22）。
- R3：提醒不重复轰炸：同一提醒实例（reminder_id）已确认后不重复弹窗。
- R4：重复日程规则正确展开，不影响非重复日程。
- R5：移动端下页面可用（列表与日历均可查看）。
