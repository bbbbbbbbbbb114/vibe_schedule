# 需求变更执行记录（步骤1-4）

## 步骤1：需求澄清与验收标准

### 目标A：强提醒（P0）
- 触发条件：日程到期时（包含即将到期窗口内）触发提醒。
- 展示方式：0级弹窗（全屏遮罩 + 居中高层模态）。
- 视觉要求：高对比警示色（红/橙）+ 大标题 + 明显确认按钮。
- 行为要求：
  - 不自动关闭。
  - 必须点击“我已确认”才能关闭。
  - 弹窗存在期间，不允许通过点击背景关闭。
- 声音要求：弹窗打开时循环提示音，点击确认后停止。

### 目标B：日历可视化（P1）
- 本轮仅完成方案与接口准备，不在步骤1-4中实现UI。

## 步骤2：技术方案

- 前端框架：保持原生 HTML/CSS/JS，减少改造成本。
- 强提醒组件：
  - 在主页面新增 `reminder-modal` 和遮罩层。
  - 使用超高 `z-index` 作为0级弹窗。
- 声音方案：Web Audio API 生成提示音（无需额外静态音频文件）。
  - 优点：本地即用、部署简单。
  - 备注：浏览器可能要求用户先有交互后允许播放声音，页面交互后可正常鸣响。

## 步骤3：数据与接口补充

- 接口沿用：`GET /api/reminders/live`。
- 前端补充策略：
  - 建立“已确认提醒集合”，避免同一提醒重复弹窗。
  - 建立提醒队列，支持同一轮多提醒顺序确认。
  - 保留轮询机制，持续获取新提醒。

## 步骤4：实现结果（本次已落地）

- 已实现0级强提醒弹窗。
- 已实现高对比警示视觉样式。
- 已实现循环提示音。
- 已实现“必须点击确认后关闭”。
- 已实现提醒去重与队列确认。
