# 需求变更（第二轮）步骤1-4执行记录

## 步骤1：需求冻结与验收标准

### 新增能力
1. 当前时间红线（本轮先完成后端准备，前端红线放在后续步骤）。
2. 提醒类型：提前 5 分钟 / 10 分钟 / 1 天。
3. 提醒时机：开始提醒 / 结束提醒 / 同时提醒。
4. 重复规则：不重复 / 每天 / 每周固定星期。
5. 地点标签：选填。

### 本轮（步骤1-4）验收口径
- 后端模型与接口支持上述字段。
- 提醒引擎按“提前量 + 时机 + 重复实例”生成实时提醒。
- 保持旧数据兼容，不破坏已有功能。

## 步骤2：数据模型扩展
- `location`: 地点标签，字符串，选填。
- `reminder_offsets`: 提醒提前量（如 `5m,10m,1d`）。
- `reminder_phase`: `start/end/both`。
- `repeat_type`: `none/daily/weekly`。
- `repeat_weekdays`: 每周重复日（0-6，周一到周日）。

## 步骤3：接口改造
- `POST /api/schedules` 与 `PUT /api/schedules/:id` 支持新字段。
- 输入校验：
  - 提醒提前量格式校验（Xm/Xd）。
  - 提醒时机与重复类型枚举校验。
  - 周重复日期范围校验（0..6）。

## 步骤4：提醒引擎升级
- `/api/reminders/live` 改为根据配置计算触发时刻：
  - 时间点：基于 `due_at`。
  - 时间段：基于 `start_at/end_at`。
  - 重复：按 `daily/weekly` 展开实例。
  - 触发：`phase_time - offset` 落入轮询窗口即返回。
- 返回 `reminder_id`，用于前端去重确认。
