# 本地部署与演示指南

## 1. 环境要求
- Python 3.11+
- Windows PowerShell / VS Code 终端

## 2. 本地部署步骤
在项目根目录执行：

```powershell
python run.py
```

启动成功标志：
- 终端出现 `Running on http://127.0.0.1:5000`

浏览器访问：
- http://127.0.0.1:5000/login

## 3. 演示脚本（课堂可直接照着说）
1. 打开登录页，输入用户名登录。
2. 进入主页面，新增一条日程：设置类型（时间点/时间段）、地点、提醒提前量（5m/10m/1d）、提醒时机（开始/结束/都提醒）、重复规则（不重复/每天/每周）。
3. 在“我的日程”与“可视化日历日程表”区域同时展示新增结果。
4. 点击“编辑”修改标题，确认列表更新。
5. 勾选完成状态，确认样式变化（删除线）。
6. 点击日历中的事件块，弹出详情信息（含地点）。
7. 点击“删除”并确认，查看列表和日历中记录同时移除。
8. 观察可视化日历中的红色“当前时间线”。
9. 新建一条到期时间很近的日程，等待 0级提醒弹窗（声音单次播放，必须确认关闭）。
10. 新建一条已过期日程（手工改早于当前时间），刷新后在“过期提醒”中可见。

## 4. 演示重点（对应验收）
- CRUD：新增/查询/编辑/删除/完成。
- 实时提醒：页面打开期间每 30 秒轮询提醒，支持提前提醒与开始/结束提醒，触发 0级弹窗。
- 过期提醒：登录后页面展示过期未完成日程。
- 可视化日历：支持当前时间红线、时间点/时间段展示、重复规则展开显示。
- 日程配置：支持地点标签（选填）、提醒类型、重复规则。
- 本地部署：SQLite 本地文件数据库，无需外部服务。
