# 要求
学习vibe coding的使用，自己给自己设定一个开发目标（如下），用vibe coding的方式加以实现。
## 开发目标
实现一个面向对用户的日程管理web应用。除了提供日程的增删改查外，还要支持日程提醒（只需支持在用户的日程管理页面处于打开状态时有实时提醒；用户登录时，可以看到过期的提醒）
## 验收要求
1. 介绍选择目前你使用的IDE的理由（很重要）
2. 介绍选择目前你使用的Python开发框架的理由（很重要）
3. 完成应用开发，并在本地部署（可以在本地环境运行并演示）
4. 介绍这个应用的功能（不重要）
5. 介绍这个应用的开发全过程（如何与AI协同，希望手写的代码越少越好）（很重要）
6. 介绍这个应用的测试过程
7. 完成上述功能后，老师会提出一次“需求变更”
8. 在现有代码的基础上针对“需求变更”完成修改，并演示
9. 介绍这个应用的回归测试过程

## 要求
1. 基于本地IDE（如：VSC、OpenCode等）与AI配合完成开发，不能使用在线的集成开发环境（如：bolt.new）
2. 数据库用SQLite
3. 编程语言用Python

## answers
1. 选择目前使用的IDE的理由:应用涉及 Python（后端）、SQLite（数据库）和 HTML/JS（前端）。VS Code有丰富的插件生态, 在不同语言之间的切换更加方便.而且 GitHub Copilot 对于认证学生免费,是性价比最高的接触顶级大模型的方式.符合项目的“本地化”要求
2. 选择目前使用的Python开发框架的理由:与SQLite天然适配,架构灵活，不会过度工程化.且Flask教程、脚手架示例、常见问题解决方案非常多，AI生成代码的可用率高，能显著减少手写代码量。对于课堂验收场景,依赖少、启动简单，flask run即可演示.
